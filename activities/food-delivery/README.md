# 🍔 Activity 1: Food Delivery System

## 🎯 Objective

Practice modeling a simple system using **UML Class Diagrams** and translating it into code.

---

## 📌 Scenario

You are building a simple Food Delivery system.

The system includes:
- A Restaurant
- Menu items offered by the restaurant

---

## 🧠 Your Task

### Step 1: Identify the Classes
Think about:
- What are the main entities in this system?

---

### Step 2: Define Relationships
- How are the classes connected?
- How many MenuItems can a Restaurant have?

---

### Step 3: Draw UML Diagram
Your diagram should include:
- Class names
- Attributes (basic ones are enough)
- Methods (simple behavior)
- Relationship between classes
- Multiplicity (important)

---

### Step 4: Implement in Java
- Create the classes
- Add attributes and methods
- Represent the relationship correctly

---

## 🚫 Restrictions

- Do NOT add unnecessary classes
- Do NOT over-engineer the solution

---

## ✅ Expected Outcome

- A clear UML diagram
- Clean and simple Java implementation

---

## ⭐ Bonus (Optional)

- Add a method to display the menu
- Add simple interaction between objects
