# 🛒 Activity 2: Online Store System

## 🎯 Objective

Apply everything learned in this lab:
- UML Class Diagrams
- Relationships
- Clean class design

---

## 📌 Scenario

You are designing a basic Online Store system.

The system includes:
- Customer
- Product
- Shopping Cart

---

## 📏 Requirements

- A Customer has exactly **one Shopping Cart**
- A Shopping Cart can contain **multiple Products**

---

## 🧠 Your Task

### Step 1: Identify Classes
You should at least have:
- Customer
- ShoppingCart
- Product

---

### Step 2: Define Relationships

Think carefully:
- What type of relationship exists between:
  - Customer and ShoppingCart?
  - ShoppingCart and Product?

- What is the multiplicity?

---

### Step 3: Draw UML Diagram

Your diagram must include:
- Class structure (attributes + methods)
- Relationships between classes
- Multiplicity
- Proper UML notation

---

### Step 4: Implement in Java

- Create all classes
- Apply relationships correctly
- Keep the design clean and readable

---

## 💡 Hints

- A cart belongs to exactly one customer
- A cart holds multiple products
- Focus on **ownership and lifecycle**

---

## ✅ Expected Outcome

- Well-structured UML diagram
- Correct implementation of relationships
- Clean, readable Java code

---

## ⭐ Bonus (Optional)

- Add:
  - `addProduct()` method
  - `removeProduct()` method
  - Total price calculation

---

## 🧩 Challenge Extension (Optional)

- Can multiple customers share the same product?
- What happens if a cart is deleted?

Think about design decisions.
